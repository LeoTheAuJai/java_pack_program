
# Homework 5 - Item Management System (Based on HW4)

## Purpose
Extend Homework 4 by refactoring the managed data into an `Item` class, demonstrating how to handle more complex data types.

## Features
- Core structure inherited from Homework 4.
- Data managed as `Item` objects (instead of primitive types or simple strings).
- Demonstrates encapsulation and object-oriented data handling.

## Key Concepts
- Creating and using custom data types (`Item`)
- Encapsulation (getters, setters)
- Object-oriented data management

# Homework 5 - 物品管理系統（基於 Homework 4）

## 目的
擴充 Homework 4，將管理的資料重構為 `Item` 類別，展示如何處理更複雜的資料型態。

## 功能
- 繼承 Homework 4 的核心結構。
- 將資料以 `Item` 物件（而非基本型別或簡單字串）進行管理。
- 展示封裝性與物件導向的資料處理方式。

## 學習重點
- 自訂資料型別 (`Item`) 的建立與使用
- 封裝（getter、setter）
- 物件導向的資料管理
