# Homework 4 - Regular Database (Predecessor of HW5)

## Purpose
Build a more structured database system, serving as the foundation for Homework 5. This version focuses on standardizing data storage and access.

## Features
- Formal database structure (compared to HW3).
- Improved data management logic.
- Designed to be extensible for future enhancements.

## Key Concepts
- Object-oriented database design
- Separation of concerns (data model vs. business logic)
- Preparation for inheritance and polymorphism

# Homework 4 - 正規資料庫（Homework 5 的前身）

## 目的
建立一個更具結構的資料庫系統，作為 Homework 5 的基礎。此版本專注於標準化資料儲存與存取方式。

## 功能
- 相較於 HW3，採用更正式的資料庫結構。
- 改善資料管理邏輯。
- 設計上保留擴展性，以便後續功能增強。

## 學習重點
- 物件導向的資料庫設計
- 關注點分離（資料模型 vs. 業務邏輯）
- 為繼承與多型做準備
